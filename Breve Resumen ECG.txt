# Simulador de ECG en ESP32 (250 Hz) + Dashboard MQTT

Este proyecto simula una señal ECG relativamente realista (P–QRS–T) en una ESP32 y envía las muestras a un broker MQTT casi en tiempo real. Un dashboard HTML ligero se suscribe al tópico y grafica la señal en el navegador.

Básicamente: un corazón falso con WiFi.

## Qué hace
- Genera muestras de ECG a 250 Hz con un modelo de morfología P/QRS/T + deriva de línea base + ruido muscular
- Usa un buffer circular para evitar “saltos” por latencia/jitter de MQTT
- Publica los datos en lotes para que la gráfica sea más estable
- Recibe comandos por MQTT para encender/apagar y cambiar BPM

## Tecnologías
- ESP32 + PlatformIO (framework Arduino)
- Tareas FreeRTOS (tarea de muestreo + tarea MQTT)
- MQTT con PubSubClient
- Dashboard web (HTML + JS) para graficar las muestras

## Tópicos MQTT
- `pacemaker/ecg`  
  Formato del payload: `timestamp,value1,value2,...,valueN`
- `pacemaker/state`  
  Publica el estado (ej.: `ACTIVO`, `INACTIVO`)
- `pacemaker/cmd`  
  Comandos:
  - `ON`
  - `OFF`
  - `BPM:<número>` (ejemplo: `BPM:75`)

## Muestreo y envío
- Frecuencia de muestreo: 250 Hz (cada 4 ms)
- Envío por lotes: ~cada 200 ms (normalmente 50 muestras)
- El buffer circular protege el muestreo de la latencia de red

## Cómo usarlo
1. Abre el proyecto en PlatformIO.
2. Configura tu WiFi en el código.
3. Compila y sube a la ESP32.
4. Abre el dashboard HTML (archivo local o hosting estático).
5. Asegúrate de usar el mismo broker y tópico.

## Notas / limitaciones
- Esto es un simulador para visualización/pruebas. NO es un dispositivo médico.
- Si el eje Y del dashboard está muy ajustado, la señal puede “cortarse”.
- Si ves la señal “plana” o “recortada”, suele ser tema de escala/límites, no del generador.

## Mejoras futuras
